# Формат файлов городов

Городские данные теперь лежат в отдельных папках по контекстам:

- `cities_mahjong/`
- `cities_test/`

Это важно, потому что один и тот же город может существовать в разных контекстах с разными игроками, клубами и рейтингами.

Каждый `*.json` внутри такой папки описывает один город для одного контекста.

Пример:

```json
{
  "name": "Москва",
  "auto_show_members": false,
  "users_text": "Необязательный текст для раздела игроков",
  "join_error_text": "Необязательный текст, запрещающий выбрать город через /my_city",
  "clubs": [
    { "title": "Фурин", "url": "https://t.me/furin_mahjong", "visible": true }
  ],
  "ratings": [
    { "title": "Рейтинг города", "url": "https://example.com/rating", "visible": false }
  ],
  "players": [
    {
      "user_id": 123456789,
      "display_name": "Игрок без username",
      "note": "необязательная подпись"
    },
    {
      "username": "nickname",
      "display_name": "Игрок с username"
    }
  ]
}
```

Поля:

- `name`: отображаемое имя города.
- `auto_show_members`: если `false`, после `/my_city` бот не будет сразу раскрывать список игроков.
- `users_text`: текст-замена для раздела игроков.
- `join_error_text`: если задан, бот не даст выбрать этот город через `/my_city`.
- `clubs`: кнопки со ссылками на клубы.
- `ratings`: кнопки со ссылками на рейтинги.
- `players`: вручную добавленные игроки для конкретного контекста.
- `visible`: необязательный флаг для `clubs` и `ratings`, позволяет быстро скрыть ссылку без удаления.

Как бот работает с игроками:

- если игрок регистрирует город через `/my_city`, бот сразу редактирует соответствующий файл в `cities_<meta>/`;
- если игрок меняет город, бот удаляет его из старого файла города и добавляет в новый;
- при миграции текущие игроки уже раскладываются по `players` внутри `cities_<meta>/`;
- если игрока нет в текущем чате, бот покажет кэшированный профиль из файла города вместо голого `ID`;
- если у игрока есть `username`, кнопка откроет `https://t.me/username`;
- если есть только `user_id`, бот использует `tg://user?id=...`, что обычно открывает профиль в Telegram-клиенте, но поведение зависит от клиента и ограничений Telegram.
